# Multi AI Agent Hub

Chrome MV3 扩展插件，支持在一个统一的界面输入文本和图片，同时向多个 AI 平台（DeepSeek, 豆包, Kimi）发送请求，并在同一个窗口中等待、收集和展示所选平台生成的结果。

## 特性

- **并行执行**：一次输入，多个 Agent 并行处理。
- **自定义选择**：通过 UI 开关灵活选择所需下发的 AI 平台。
- **多模态支持**：支持附加本地图片并上传至启用了识图能力的平台。
- **自动收集结果**：自动监听各个页面的生成状态，将完成后的文本和生成的图片抓取回显。

## 安装步骤

1. 打开 Chrome 浏览器，进入 "管理扩展程序" 页面 (`chrome://extensions/`)
2. 开启右上角的 "开发者模式"
3. 点击 "加载已解压的扩展程序"
4. 选择本项目根目录（包含 `manifest.json` 的目录）

## 使用方法

由于扩展通过向特定网页注入 Content Script 来完成自动化操作，**如果未登录对应的 AI 服务将会导致执行失败**。

1. **预先登录**：
   - 请在浏览器中提前打开并登录以下页面：
   - DeepSeek: [https://chat.deepseek.com](https://chat.deepseek.com)
   - 豆包: [https://www.doubao.com](https://www.doubao.com)
   - Kimi: [https://kimi.moonshot.cn](https://kimi.moonshot.cn)
2. **交互**：
   - 点击浏览器工具栏的插件图标打开 Popup 界面。
   - 输入您要提问的文本或 Prompt。
   - （可选）从本地选择一张图片上传。
   - 使用开关勾选希望参与本次问答的 AI。
   - 点击 "提交到选中 Agent"。
3. **查看结果**：
   - 提交后，插件会自动切换或打开相关页面的标签，并开始自动录入和发送。
   - 请稍作等待，面板中各 Agent 的卡片状态会从 "执行中" 变为 "已完成"，并展示其回答。

## 注意事项

- 本插件依赖于目标网站 DOM 结构的解析（强依赖页面元素选择器）。如果目标平台的主题、版式发生变动，可能会导致部分功能或监听失败，需要及时更新 `agents` 目录内的适配器选择器配置。
